The first new service you will be learning is the File Transfer Protocol (FTP).

There are multiple different applications that use ftp. The one that we will be using is vsftpd which is not yet installed on the computer.

Install it using the command "sudo apt install vsftpd" which will prompt you for your password. Like in the last module, you will not be able to see characters being typed.

VSFTPD installed ❌

If the command does not work, try running "sudo apt update" first and then run the previous command.

Manuver to "/etc" and edit the file "ftpusers" to include the users "root" and "developer" in them. Adding users to this file prevents them from using ftp on the system. We never want root to be able to use ftp as it gives a user too

Now begin editing the file vsftpd.conf.

To stop anonymous connections, change "anonymous_enable=YES" to "anonymous_enable=NO" which is require all users to know an authorized username and password. This prevents unauthorized users from having access to your system which ftp can allow.

Anonymous mode disabled ❌

Next we will cover chroot. In the context of ftp, chroot prevents users from accessing certain files/directory while accessing the computer using ftp. To enable this function in ftp, change "chroot_local_enable=YES"

Chroot enabled ❌

To make the chroot effective, we must setup the directory to be used as a jail. Luckily, I have already done this. All you need to do is set "secure_chroot_dir" equal to "/ftp/ftpuser/jail"

Test the chroot jail by running the command "ftp localhost" and then entering the credentials of the user "ftpuser" with the password "l0nestar" to login. Using cyber will not work as it is configured for ftp to not chroot them.

Attempt to switch directories. It should throw an error at you as you do not have access to any other directory. There are two files within your current folder, test and a hidden file. An important part of cyber is being able to use google. Find out "how to see hidden files in Linux terminal."
